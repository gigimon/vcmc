# Step 13: Стабильный табличный layout

## Цель
Сделать рендер списка файлов стабильным по геометрии, чтобы длина имени файла не влияла на ширины колонок.

## Решения
- Перешли на fixed-колонки от ширины панели (без зависимости от контента):
  - full layout: `Name 64% | Size 14% | Modified 22%`
- Добавлен narrow fallback:
  - `Compact` (Name + Size) при узкой панели
  - `Minimal` (только Name) при очень узкой панели
- Обрезка значений (включая дату/размер) унифицирована через `truncate_name`.
- Заголовок таблицы и строки используют один и тот же layout-расчет.
